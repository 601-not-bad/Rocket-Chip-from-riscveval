Declaration of 'virtual' Memory components.

Member Data Documentation

BootRom_inferred
Component

Declaration of the BootRom component used for RTL simulation.

This component is also valid for the FPGA implementation so that it uses standard HEX file for the ROM initialization.

BootRom_tech
Component

Declaration of the "virtual" BootROM component.

BootRom start address must implements address matching to the CPU reset vector (0x200) and all processing after power-on is using this memory block. BootRom size depends of the configuration and size of the generated hex file. Component implements one-clock access to the ROM without wait-staits. Datawidth depends of the AXI4 bus configuration.

Parameters
[in]techGeneric technology selector.
[in]hex_filenameGeneric argument defining hex-file location.
[in]clkSystem bus clock.
[in]addressInput address.
[out]dataOutput data value.
Ram32_tech
Component

Virtual SRAM block with fixed 32-bits data width.

This module doesn't support byte access and always implements 4-bytes alignment.

Ram64_tech
Component

Virtual SRAM block with fixed 64-bits data width.

This module doesn't support byte access and always implements 8-bytes alignment.

RomImage_inferred
Component

Declaration of the RomImage component used for RTL simulation.

This component is also valid for the FPGA implementation so that it uses standard HEX file for the ROM initialization.

RomImage_tech
Component

Declaration of the "virtual" RomImage component.

This module stores pre-built firmware image that is coping into internal SRAM during Boot stage without any modificaiton. RomImage size is limited by global configuration parameter and it cannot be more than internal SRAM size. Component implements one-clock access to the ROM without wait-staits. Datawidth depends of the AXI4 bus configuration.

Parameters
[in]techGeneric technology selector.
[in]sim_hexfileGeneric argument defining hex-file location.
[in]clkSystem bus clock.
[in]addressInput address.
[out]dataOutput data value.
RomPrn_tech
Component

Galileo PRN codes ROM storage:

This ROM is used in FSE Engine to form reference E1 reference signals. HEX-file isn't used for this ROM because 'inferred' module was built using "case when" operators.

srambytes_tech
Component

Declaration of the "virtual" SRAM component with unaligned access.

This module implements internal SRAM and support unaligned access without wait-states. For example it allows to read 4 bytes from address 0x3 for one clock. Component implements one-clock access without wait-staits. Datawidth depends of the AXI4 bus configuration.
