Empty output signals of HostIO interface.

If device was included in the owners of the HostIO interface and was disabled by configuration parameter (for example) then its outputs must be assigned to this empty signals otherwise RTL simulation will fail with undefined states of the processor.

htif_serdes_in_type
Record

HTIF serializer input.

In a case of using L2-cache, 'Uncore' module implements additional layer of the transformation of 128-bits HTIF messages into chunks of HTIF_WIDTH. So we have to implement the same serdes on upper level.

HTIF_WIDTH integer := 16
Constant

Multiplexing HTIF bus data width.

Not used in a case of disabled L2 cache. If L2 cached is enabled this value defines bitwise of the bus between Uncore module and external transievers. Standard message size for the HTID request is 128 bits, so this value defines number of beats required to transmit/recieve such message.

htifctrl
Component

HostIO (HTIF) controller declaration.

This device provides multiplexing of the Host messages from several sources (interrupt controller, ethernet MAC, Debug Support Unit and others) on HostIO bus that is specific for Rocket-chip implementation of RISC-V.
